The soft in software

We sometimes use the term 'hard coded' when writing something like const withTaxes = sum * 1.19. The tax of 19% can change and if it does, we have to find all the spots where this number was used. In some cases the code might be written like this though: const total = sum * 19 / 100. So we would miss it if we searched the document for 1.19. In this context most developers would recognize what the number 19 means. But it could also be something like this original piece of code from the source of Quake III Arena: i = 0x5f3759df - ( i >> 1 ); // what the fuck? (read more) The comment at the end is telling. The tax number is pretty easy to fix. Instead of writing the number, we use a variable. We can pass it by value in the function where it is used. If we really 'need' to, we could use a global variable, even if global variables are frowned upon. If it was at least immutable, it would be difficult to do much harm. In some cultures, devs use SCREAMING_CASE to mark globals. Then we would write like this: const withTaxes = sum * TAX_RATE. It sticks out quite a lot. Which I like. Now everyone knows we are bad developers, because we use globals ;). In a small application this could actually work ok. It might not be the best idea anywhere else. Context does matter. It's still better than writing out the number for anything but throw-away code though.

What's the point?

Well, yes. I mean, this is actually not a good example for hard challenges in larger teams, because if not one person on a team sees a problem in the use of 'magic numbers', I do not think it will survive very long. I could be wrong, but that's my guts opinion. Magic numbers are not the only and by far not the worst problem to have. They make changes hard, but it's still pretty fixable. At least in the case of simple numbers like a tax rate. The number in the fast inverse square root algorithm that ID Software came up with is also actually not a big problem as long as it is never changed and only used in one place. Things do get more difficult when the meaning of a value is not clear and even more difficult when they get used at seemingly random places in the source code. I hope you can imagine why that could cause pain.
